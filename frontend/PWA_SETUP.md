## 📦 Installation & Setup

### Step 1: Verify Packages

All required packages are already in `package.json`:

```json
{
  "dependencies": {
    "vite-plugin-pwa": "^1.2.0"
  }
}
```

If not installed, run:
```bash
cd frontend
npm install vite-plugin-pwa
```

### Step 2: Generate Icons

This is **crucial** for PWA functionality. Follow one of these methods:

#### **Quick Method: Online Tool**
1. Go to: https://icoconvert.com/
2. Upload `frontend/public/image.png`
3. Download PNG versions for: 96px, 192px, 512px
4. Save to `frontend/public/icons/` with these names:
   - `aura-icon-96.png`
   - `aura-icon-192.png`
   - `aura-icon-512.png`
   - `aura-icon-192-maskable.png`
   - `aura-icon-512-maskable.png`
   - `aura-screenshot-192.png`
   - `aura-screenshot-512.png`

#### **Automated Method: Node.js Script**
```bash
cd frontend
node generate-icons.js
```

**Note:** This requires ImageMagick or Sharp to be installed.

See `ICON_GENERATION_GUIDE.md` for detailed instructions.

## 🎨 Icon Generation

### Quick Steps

1. **Ensure `image.png` exists in `frontend/public/`**

2. **Generate icons** (choose one):

   **Option A: Online Tool (Easiest)**
   - Visit https://icoconvert.com/
   - Upload image.png

   - Manually place in public/icons/

   **Option B: Automated Script**


   **Option C: Manual ImageMagick Commands**
   ```bash
   mkdir -p frontend/public/icons
   convert frontend/public/image.png -resize 192x192 -background white frontend/public/icons/aura-icon-192.png
   convert frontend/public/image.png -resize 512x512 -background white frontend/public/icons/aura-icon-512.png
   # ... (see ICON_GENERATION_GUIDE.md for all commands)
   ```

3. **Verify generated files**:
   ```bash
   ls -la frontend/public/icons/
   ```

### Icon Requirements

- **Format**: PNG with transparency support
- **Sizes**: 96×96, 192×192, 512×512 pixels
- **Colors**: Support your app's color scheme
- **Quality**: At least 300 DPI equivalent

## 🌍 Deployment

### Step 1: Build Locally & Test
```bash
cd frontend
npm run build
npm run preview
```

### Step 2: Deploy to Production

Ensure your deployment supports:
- ✅ **HTTPS** (Required for Service Workers)
- ✅ **Correct MIME types** for `.js`, `.css`, `.wasm` files
- ✅ **Caching headers** for assets
- ✅ **Cache-busting** for `index.html`

### Step 3: Popular Deployment Platforms

#### **Vercel** (Recommended)
```bash
npm install -g vercel
cd frontend
vercel --prod
```

#### **Netlify**
- Connect your Git repo
- Build command: `npm run build`
- Publish directory: `dist`

#### **GitHub Pages**
```bash
cd frontend
npm run build
# Deploy dist/ folder to GitHub Pages
```

#### **Traditional Server** (Nginx/Apache)
```bash
# Copy dist/ folder to server
scp -r frontend/dist/ user@server:/var/www/aura/

# Nginx config
server {
    listen 443 ssl http2;
    server_name aura.example.com;
    
    root /var/www/aura;
    index index.html;
    
    # Cache static assets
    location ~* \.(js|css|png|jpg|svg|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # Don't cache index.html or service worker
    location ~ (index.html|sw.js) {
        expires -1;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
    
    # Fallback to index.html for SPA routing
    try_files $uri /index.html;
}
```

### Step 4: Verify Deployment

After deployment, check:

```bash
# Check HTTPS
curl -I https://your-aura-domain.com

# Should return HTTP/2 and proper headers
# Verify Service Worker
curl -I https://your-aura-domain.com/sw.js

# Verify manifest
curl https://your-aura-domain.com/manifest.json

# Check PWA on staging/production
# Open on different devices and test install prompts
```

## 🐛 Troubleshooting

### Service Worker Not Registering

**Problem**: No service worker in DevTools

**Solutions**:
1. Check browser console for errors
2. Ensure site is served over HTTPS (or localhost)
3. Clear cache: DevTools → Application → Clear all
4. Rebuild: `npm run build`

```javascript
// Check in console
navigator.serviceWorker.getRegistrations()
  .then(regs => console.log('SWs:', regs))
```

### Install Button Not Showing

**Problem**: "Install App" button doesn't appear

**Solutions**:
1. Must be HTTPS (except localhost)
2. Manifest must be valid (check DevTools)
3. Icons must exist and be proper size
4. Check `beforeinstallprompt` in console:
   ```javascript
   window.addEventListener('beforeinstallprompt', (e) => {
     console.log('Install prompt available!');
   });
   ```

### Icons Not Showing

**Problem**: App icons appear broken or missing

**Solutions**:
1. Verify files exist: `ls -la frontend/public/icons/`
2. Check icon paths in `vite.config.js` match actual files
3. Icons must be valid PNG files
4. Clear browser cache and rebuild

```javascript
// Debug manifest in console
fetch('/manifest.json').then(r => r.json()).then(m => console.log('Manifest:', m))
```

### Offline Page Shows Wrong Content

**Problem**: Offline fallback page displays but UI looks broken

**Solutions**:
1. This is normal - offline page is simple HTML
2. Check `public/sw.js` for the offline HTML
3. Customize offline page by editing `createOfflineResponse()` function

### PWA Won't Update

**Problem**: Old version keeps showing after update

**Solutions**:
1. Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
2. Manually dismiss update banner to reload
3. Uninstall app and reinstall
4. Clear all caches:
   ```javascript
   // In console
   caches.keys().then(names => 
     Promise.all(names.map(name => caches.delete(name)))
   )
   ```

### API Calls Fail Offline

**Problem**: API errors show when offline

**Solutions**:
1. This is expected - offline fallback shows limitations
2. Check cached responses in DevTools → Application → Cache Storage
3. Verify API URLs in `vite.config.js` runtime caching config
4. Consider implementing queue for offline requests

## 📊 Monitoring & Analytics

### Storage Usage
```javascript
// Check storage usage
if ('storage' in navigator && 'estimate' in navigator.storage) {
  navigator.storage.estimate().then(est => {
    console.log(`Used: ${est.usage} bytes`);
    console.log(`Available: ${est.quota} bytes`);
    console.log(`Percentage: ${(est.usage/est.quota*100).toFixed(2)}%`);
  });
}
```

### Service Worker Status
```javascript
// Check SW status
navigator.serviceWorker.getRegistrations().then(regs => {
  regs.forEach(reg => {
    console.log('SW State:', {
      installing: reg.installing?.state,
      waiting: reg.waiting?.state,
      active: reg.active?.state,
    });
  });
});
```
